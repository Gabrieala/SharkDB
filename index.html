Welcome to the SharkDB Scientific Terminal

Our oceans house **over 500+ known species of sharks**, ranging from tiny lanternsharks to the massive whale shark. Oceanographic expeditions regularly discover new species, especially in the deep sea. SharkDB provides tools to explore all cataloged species and consult our RAG agent to analyze their ecology.

Scientific Data Integrity

Our verified profiles are compiled from Wikipedia, the Shark Research Institute, and WWF. For species not pre-loaded locally, SharkDB triggers **real-time Wikipedia API queries** to retrieve summaries instantly.

The Uncharted Ocean & Future Discoveries

Marine biologists estimate that hundreds of deep-sea shark species remain unclassified. In the last decade alone, discoveries like the Ninja Lanternshark (*Etmopterus benchleyi*) have shown how little we know about deep ocean layers. SharkDB supports expanding its library dynamically via internet searches to catalog these newly discovered species as research advances.

Just type any shark species name in the search bar below (e.g. "Epaulette shark", "Goblin shark", "Ninja lanternshark"). If it's not pre-loaded locally, SharkDB will automatically search Wikipedia and insert the complete biological profile into your catalog.
Out of scope: Only members of the Selachimorpha (sharks) class can be cataloged.

SharkDB AI Architecture & Pipeline

This web application showcases a client-side **Retrieval-Augmented Generation (RAG)** pipeline designed to explore shark biology without hallucinations or factual errors.

1. Structured Ground Truth Database

Instead of letting the language model hallucinate statistics and parameters, we define a structured local database in js/database.js. This database holds verified scientific parameters (average and maximum length, maximum weight, speed, habitats, threat levels, conservation status, behaviors, and diets) sourced from:

  • Wikipedia: Common names, scientific classifications, general descriptions, and behavioral contexts.
  • Shark Research Institute (SRI): Detailed physiological dimensions, speed benchmarks, and records on human safety levels.
  • World Wildlife Fund (WWF): Accurate conservation status indicators (Endangered, Vulnerable, CITES status) and habitat preservation contexts.

2. Wikipedia API Scalability (500+ Species)

To support all 500+ known species of sharks, SharkDB incorporates a **Wikipedia Real-Time Retrieval Module**. When you search or ask about a non-cataloged species (e.g. "Epaulette shark"):

  1. The system queries the Wikipedia Search API to locate the exact taxonomic title.
  2. It retrieves the page summary, scientific binomial names, thumbnail image, and main paragraphs in real time.
  3. It dynamically constructs a formatted SharkDB record containing estimated sizes, velocities, habitats, and safety parameters.
  4. This dynamic context is then injected directly into the active RAG engine (local template or Gemini model).

3. RAG & Semantic Search (TF-IDF)

Upon initial application load, the system builds an index using the Term Frequency-Inverse Document Frequency (TF-IDF) formula across all species profiles:

TF(t, d) = (Number of times term t appears in document d) / (Total terms in d)
IDF(t) = log(1 + Total Documents / Documents containing term t)
TF-IDF Weight = TF * IDF

When you ask a question in the chatbot interface, the system tokenizes the query, eliminates standard grammatical stopwords, computes the cosine similarity vector, and retrieves the top 3 most semantically matching shark records as "Context Records".

4. Flexible AI Reasoning Modes

  • Gemini 2.5 Flash RAG (API Key Mode): If a Google Gemini API Key is entered, the app makes an HTTPS request to the official Gemini API. The prompt is wrapped with strict instructions forcing the model to rely only on the retrieved database records for context, citing sources, and responding with "I'm sorry, but that information is not available..." if it goes outside the bounds of the database.
  • Local Expert Engine (Offline Mode): If no API key is provided, the client-side system acts as a highly structured local NLP engine. It parses common query intents (comparisons, danger metrics, speeds, sizes, species checks) using regex parameters, references the matching database records directly, and formats structured responses (like side-by-side markdown comparison tables) with zero lag and zero hallucination.

5. Profile Portability (JSON Backups)

To keep the application deployable as a static asset without database backend costs, SharkDB runs localized user sessions in the browser. To allow portability, we implement a **JSON Backup/Restore tool**. Users can download a JSON backup of their account profile, bookmarks, and chat logs, and upload it on any other deployed device to sync their session.
